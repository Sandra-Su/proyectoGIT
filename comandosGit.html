Intruccion Descripcion
git --version Nos muestra la version que tenemos instalada
git init inicializa el repositorio
git add Añade los archivos que hemos modificado o creado. Se puede especificar el nombre del archivo o poner un punto despues del add para que agregue todo
git commit -m "mensaje" Añade un comentario que explica los cambios que acabamos de hacer con el comando git add
git commit -am "mensaje" Junta la función del comando git add junto con git commit
git config –global user.name “nombre” Nos permite definir el nombre de nuestro usuario, para poder realizar los commits
git config –global user.email “correo electrónico” Nos permite definir el correo de nuestro usuario
git log nombreArchivo.ext Nos muestra los commits con su respectivo id También podemos escribir solo git log
git show nombreArchivo.ext Nos muestra lo que hemos quitado y agregado, es decir, las modificaciones que se le hicieron al archivo
git commitA commitB muestra las diferencias entre ambas versiones. A este comando se le mandan los id de los commit que se quieren comparar
git reset Nos ayuda a volver en el tiempo sin necesidad de ver el historial
  • -hard borra todo, es una mala práctica
  • -soft vuelve a la versión en cuestión pero respeta lo que hay en el área de staging
  • -HEAD saca los archivos del área de staging
git rm Elimina archivos de git sin eliminar su historial del sistemas de versiones. Buena práctica
  • -cached elimina los archivos del área de staging y del próximo commit pero los mantiene en nuestro disco duro
  • -force elimina los archivos de git y del disco duro
git branch nombreRama Crea una nueva rama. Si solo ponemos git branch nos mostrará todas las ramas que tenemos
git checkout nombreRama Nos permite movernos a la rama que deseamos
git merge nombreRama Nos ayuda a mezclar las ramas y crear una versión final. El nombre de la rama es de donde se extraerán los cambios
